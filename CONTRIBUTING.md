# Contributing

Thanks for your interest in contributing!

## Getting Started

- Fork the repository and create a new branch from `main`
- Install dependencies and read `docs/ENVIRONMENT_SETUP.md`
- Use `.env.example` to prepare your local `.env.local`

## Development Standards

- TypeScript strictness: keep types explicit for public APIs
- Lint before committing: `npm run lint`
- Keep edits focused; avoid unrelated refactors

## Commit Messages

- Use conventional commits where possible, e.g.:
  - feat: add X
  - fix: correct Y
  - docs: update Z

## Pull Requests

- Link related issues
- Include screenshots for UI changes
- Describe testing performed and edge cases considered

## Security

- Do not include secrets in PRs
- Report vulnerabilities privately to security@yourdomain.example
